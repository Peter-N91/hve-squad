Pick a Squad Demo

Each demo is a runnable script that drives one workload end to end from the single /squad entry point — research → plan → implement → review underneath, with a human gate on every action that matters. Choose a chevron above, or pick a card below; the walkthrough opens on its own page so you can run it beat by beat.

The demos

Demo 1

Azure Squad

One Azure workload from cost estimate through architecture, IaC, a governance-checked deploy, as-built documentation, and diagnosis. MCP-optional with graceful fallbacks; every deploy stays behind a human gate; as-built and diagnose are strictly read-only.

Technical · Azure · profile=azure
Demo 2

Migration Autopilot

A single autopilot request plans a Contoso Inc. → Azure migration end to end — research, plan, council, Bicep IaC, cost, security, and HLD/LLD diagrams — then the review gate catches five production blockers and stops, escalating instead of shipping broken work.

Technical · Migration · mode=autopilot
Demo 3

Product Squad

One business idea from discovery through a requirements document, a roadmap and backlog, a cheap validation experiment, a readiness review, and a board-ready deck. No cloud, no code, no spend — built to pitch to a non-technical audience.

Non-technical · Product · profile=product
Demo 4

Modernize .NET

A single autopilot request upgrades a legacy app in place from .NET Framework 4.8 to .NET 10 — research, a pre-implementation council, a security Risk Gate a human approves, the SDK-style upgrade, and a review loop that remediates every finding to zero before Go.

Technical · Modernization · mode=autopilot

How to read a demo

Both demos share the same shape, so once you have run one the others feel familiar:

SectionWhat it gives you
What you are provingThe narration backbone — each beat mapped to the squad role that owns it.
Before you startPrerequisites and the safe, zero-spend way to run it live.
Beat 0 → Beat NThe runnable /squad requests, in order, with what to narrate at each.
The one-request versionThe same arc as a single autopilot run that stops only where it must.
Failure-safe fallbacksWhat to do when tooling hiccups, so the narrative never breaks.

Same methodology, different audience. The Azure demo proves the squad on infrastructure; the Product demo proves the same research → plan → build → review spine on non-technical business work. Pick the one that matches who is in the room.
